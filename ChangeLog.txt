Μεταβλητές που προστέθηκαν:
	map<string, string> controUnit:
		Η μεταβλητή αυτή ειναι υπεύθυνη για να χειρίζεται όλες αλλαγές στην κατάσταση του αντικειμένου που ειναι ενεργό τη στιγμή αυτή. 
		Π.χ. οταν ο παίχτης ειναι στο Main Menu το αντικείμενο που ειναι ενεργό ειναι το mainmenu και το controUnit παρακολουθεί την εκτέλεση του (initialization, running, next state κτλ)
		Η μεταβλητή αυτή ειναι map παρακάτω παραθέτω τα βασικά ζευγάρια key|value που εχω συμπεριλάβει ως τωρα στην map	
			- "Initialized"|"true" ,""
			- "Running"|"true","false"
			- "Next State"|"","<The next Engine::State>"

Αναβάθμιση Engine:
	Το ControUnit θα πρεπει να περνάει με αναφορά σε όλες τις κλάσεις στις οποίες μας ενδιαφέρει η κατάσταση στην οποία βρίσκονται (Μάχη, CutScene κτλ)
	Ιnitialization phase Αρχικά το main game loop μπαίνει στο (βλεπε κώδικα) εκεί καλεί την αντίστοιχη μέθοδο init της κλάσης που θέλει να ξεκινήσει. Θέτουμε το Initialized σε true στο ControlUnit.
	Αν με την προηγούμενη ενέργεια μας μπορούμε να καλέσουμε την update πάνω σε αυτό το αντικείμενο θέτουμε και το Running σε true
	Οταν η τελειώνει η ρουτίνα μας θέτουμε Running σε false και Initialized σε "" και Next State στο next Engine::State
	Και ξεκινάμε απο την αρχή
	
Ο λόγος που εγινε ολο αυτό:
	Το Engine πρεπει να έχει πλήρη έλεγχο πάνω σε ολα τα αντικείμενα του και να δρα παράλληλα με αυτά
	Η MainMenu ειχε καταλήξει να ειναι ενα ξεχωριστό Engine μεσα στο game μας το οποίο θα προκαλούσε ασχημα bugs σε καταστάσεις που πρέπει η MainMenu να επικοινωνήσει με το υπόλοιπο παιχνίδι (Settings, Load Game)
	Επιπλεον με την προσθήκη του ControUnit μπορούμε πλεον να μεταφέρουμε πολλαπλές μεταβλήτες μεταξύ Engine και του ενεργού αντικειμένου έτσι ώστε να μην έχουμε ενα σωρό από getter και setters οι οποίες θα κάνουν το κώδικα περίπλοκο
